# Jidouka — Record & Replay

Jidouka is a powerful Chrome extension designed to record your browser actions and replay them as automated workflows. Whether you need to automate repetitive form submissions, test web workflows, or create simple macros, Jidou andka makes it easy without writing any code.

## 🚀 Features

- **Seamless Recording**: Capture clicks, typing, selections, form submissions, and even URL navigation effortlessly.
- **Robust Replay Engine**: Uses advanced element fingerprinting to identify elements accurately, making your recorded actions resilient to minor DOM changes or re-renders.
- **Intuitive Side Panel**: Manage your recordings and execute automations through a user-friendly React-based side panel UI.
- **Smart Automation**: Automatically handles delays and waits for elements to appear during replay to ensure high reliability.
- **SPA Support**: Capable of detecting navigation in Single Page Applications (SPAs) where the URL changes without a full page reload.

## 🛠️ How It Works

Jidouka operates using three main components:

1.  **Content Scripts**: Injected into web pages to listen for user interactions (clicks, inputs, etc.) and execute recorded actions.
2.  **Background Service Worker**: Manages the state of recording and orchestration between the content scripts and the side panel.
3.  **Side Panel UI**: A React application that provides the interface for starting/stopping recordings and triggering replays.

To ensure reliability, Jidouka implements **Element Fingerprinting**. Instead of relying solely on fragile CSS selectors, it creates a unique "fingerprint" for elements (using text, aria-labels, placeholders, etc.), allowing the automation to find the correct target even if the page structure shifts slightly.

## ⚙️ Building & Installation

Since Jidouka is a Chrome extension, you need to build the project before it can be loaded into your browser.

### 🛠️ Building the Project

You can build the project in two modes:

#### 1. Production Build (Recommended for stable use)
This creates a optimized, minified version of the extension.
```bash
npm run build
```
After running this command, a `dist` folder will be created containing the files needed for the browser.

#### 2. Development Mode (For active development)
This mode uses Vite's watch mode to automatically rebuild the extension whenever you change a file.
```bash
npm run dev
```

### 🚀 Installing in Chrome

Once you have built the project (either via `build` or `dev`), follow these steps to load it into your browser:

1.  Open Google Chrome and navigate to `chrome://extensions/`.
2.  Enable **Developer mode** using the toggle switch in the top right corner.
3.  Click the **Load unpacked** button that appears in the top left.
4.  In the file picker, navigate to your project directory (`c:\sites\jidouka`) and select the `dist` folder (or the directory containing your built files).
5.  The Jidouka extension icon should now appear in your extensions list! 
6.  Click the **Jidouka icon** (or pin it from the extensions menu) to open the side panel and start automating.

### For Developers (Development Workflow)
If you want to contribute or modify the project, follow these steps:

1.  **Clone the repository**:
    ```bash
    git clone https://github.com/minh1997/jidouka.git
    cd jidouka
    ```

2.  **Install dependencies**:
    ```bash
    npm install
    ```

3.  **Run in development mode**:
    This will start the Vite watcher to rebuild the extension whenever you make changes.
    ```bash
    npm run dev
    ```

4.  **Load into Chrome**:
    Point Chrome's "Load unpacked" feature to the `dist` folder generated by your build/dev process.
